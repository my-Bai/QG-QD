# CSS3基础  

> *CSS3为最新的CSS标准，所有浏览器目前均已兼容。*  

## 组合选择器  

组合选择器是指使用组合器连接两个及以上选择器的选择器，组合器定义了元素之间的关系，该选择器分为以下四种：  

- 后代选择器  
- 子选择器  
- 相邻兄弟选择器  
- 通用兄弟选择器  

此处讲解前两种：后代选择器与子选择器。  

### 后代选择器  

又称包含选择器，组成器为空格（SPACE），可以选择一个父元素中的所有某类别子元素，无论是直接子元素还是间接子元素都包括在内，其格式为`父元素 子元素 {声明}`。  

> *注意：选择包含间接子元素在内的某类别子元素时，推荐格式为 `父元素 直接子元素 1层间接子元素 ··· n层间接子元素 {声明}`。*  

### 子选择器  

组成器为大于号（`>`），可以选择一个父元素中的某类别直接子元素，间接子元素不包含于被选中元素中，其格式为`父元素>子元素 {声明}`。  

> *注意：父元素与子元素的选择可以使用任意基础选择器，如 `.abc #def {声明}`。*  

## 选择器列表  

选择器列表是选择器的一种，可以同时选择多种元素，通常用于集体声明，元素间用逗号（`,`）隔开，格式为：  

```  
元素1,
元素2,
···,
元素n {声明}
```

## 伪类选择器  

伪类选择器用于选择具有特定状态或位置的元素，选择时应先输入一个冒号（`:`），此处讲解其中一种：动态伪类选择器。  

### 动态伪类选择器  

- `a:link`：选择所有未被访问的链接  
- `a:visited`：选择所有已被访问的链接  
- `a:hover`：选择鼠标指针位于其上的链接  
- `a:active`：选择活动链接（鼠标指针按下之后与弹起之前）  

> *注意：实际应用时为确保生效，以上四种选择器最好按照从上到下的顺序进行代码布局。*  

> *注意：由于链接`<a>`元素在浏览器中具有独立的默认样式，故链接所需样式需要单独指定。*  

- `input:focus`：选择获得焦点的表单元素  

## 盒子模型  

### 组成部分  

CSS盒子模型的主要组成成分包括边框（`border`)，内、外边距(`padding`、`margin`）与实际内容（`content`）四部分。  

> *注意：边框与内、外边距会影响盒子模型的实际宽度与高度。*  

### 关于单行文本垂直居中  

使盒子的高度等于行间距，即可使盒子中的单行文本垂直居中。  

### 关于显示模式的转换  

某些情况下，一种显示模式的元素需要另一种显示模式的特点，称为显示模式的转换，以下为三种的转换模式：  

- 转换为块级元素：`display: block;`  
- 转换为行内元素：`display: inline;`  
- 转换为行内块元素：`display: inline-block;`

### 边框    

#### 样式  

`border-style`属性定义了要显示的边框类型，以下为其属性值：  

- 无边框（默认）：`none`  
- 隐藏边框：`hidden`  

> *注意：无边框与隐藏边框都看不见边框存在，但隐藏边框占据布局空间且影响盒模型运算。*  

- 实线边框：`solid`  
- 虚线边框：`dashed`  
- 点线边框：`dotted`  
- 双边框：`double`  
- 3D边框：  
  - 坡口边框：`groove`  
  - 脊线边框：`ridge`  
  - 凹边框：`inset`  
  - 凸边框：`outset`  
  
  > *注意：3D边框兼容性严重，建议谨慎使用。*  

边框属性值可以为1~4个，顺序依次为：  

- 1个：同时定义四边。  
- 2个：先定义上下两边，再定义左右两边。  
- 3个：先定义上边，再定义左右两边，最后定义下边。  
- 4个：按照顺时针顺序从上边开始定义。  

> *注意：边框样式是所有边框属性的基础，若缺少样式则其他边框属性将不起任何作用。*  

#### 宽度  

`border-width`属性定义了边框四边的宽度，单位有像素（px），点（pt），派卡（pc），毫米（mm），厘米（cm），英寸（in）等，页面最常用单位为px，属性值可以为1~4个，定义顺序与`border-style`相同。

#### 颜色  

`border-color`属性定义了边框四边的颜色，属性值可以为1~4个，定义顺序与`border-style`相同。  

> *注意：以上三种属性还可为指定某边定义，格式为：`border-top/bottom/right/left-xxx`。*  

#### 属性简写  

如需缩短代码，可以在一个`border`属性中指定上述所有边框属性，其输入顺序为：  

1.`border-width`  

2.`border-style`  

3.`border-color`  

也可以只为一边进行属性简写，属性名格式为`border-top/bottom/right/left`

其余事项与背景属性简写相同。  

#### 表格  

`border-collapse`属性控制表格中相邻单元格边框的合并，合并时属性值应定义为`collapse`，单元格的合并指的是去除相邻边框的宽度叠加问题，使相邻单元格边框相遇后自动合并为一个边框，而非两个边框同时存在。  

> *注意：CSS定义表格边框时，选择器选中的元素应为`th`或`td`，若选中的元素为`table`，则只有整个表格的最外侧存在边框，无法起到划分单元格的作用。*  

### 内边距  

`padding`属性定义了内边距，即边框与内容之间的距离，格式为数字+单位，属性值可以为1~4个，定义顺序与`border-style`相同。    

> *注意：`padding`属性还可为指定某边定义，格式为：`padding-top/bottom/right/left`。*  

### 外边距  

`margin`用于在已定义的边框外为元素周围创建空间，属性值表示圆角半径，以数值表示时单位与`border-width`相同，属性值可以为1~4个，定义顺序与`border-style`相同。  

> *注意：以数值表示时其属性值还可以是负值，用于使元素向对应方向移动一定距离。*  

`margin`属性值也可以设置为`auto`，即浏览器自动计算外边距，使元素在其盒模型中水平居中，占据指定的宽度并将剩余空间在左右边界之间平均分配。

> *注意：`margin`属性还可为指定某边定义，格式为：`margin-top/bottom/right/left`。*   

> *注意：行内元素为照顾浏览器的兼容性，尽量只设置左右的内、外边距，不要设置上下的内、外边距。*  

### 关于内外边距  

#### 内边距与宽度、高度  

内边距与盒子模型的宽度（`width`）与高度（`height`）属性并不相容，如盒子宽度（`width`）定义为为20px，而左右内边距皆定义为为5px，则盒子模型的真正宽度应为20+5+5=30px，但若盒子模型的宽度与高度属性值未直接设置，则此时内边距不会影响该盒子模型的高度与宽度。  

#### 外边距与水平居中  

外边距可以使盒子模型在页面中水平居中，只需要满足以下两个条件：  

- 盒子模型有指定的宽度（`width`）  
- 盒子模型的左右外边距属性值均为`auto`  

#### 外边距合并  

- 上下相邻的两个盒子模型相遇时，若上面的盒子模型带有下外边距，且下面的盒子模型带有上外边距，则外边距不会像内边距一样叠加，而是以其中数值更大的一方为二者间的真实外边距。  

- 父元素与子元素若同时具有上外边距或同时具有下外边距，则会发生嵌套元素塌陷——上外边距或下外边距将会选择二者中数值更大的一方为真实外边距，同时父元素与子元素会绑定，并同时根据该外边距位移，使父元素与子元素上边框或下边框间无间隔距离。  

  > *若想要父元素与子元素上、下边框间存在间隔距离，有以下三种方法：*  
  >
  > - *为父元素定义上、下边框*  
  > - *为父元素定义上、下内边距*  
  > - *为父元素添加属性`overflow: hidden`*  

  > *注意：浮动，固定或绝对定位的盒子模型不会发生嵌套元素塌陷。*  

#### 清除内外边距  

很多网页元素带有默认的内外边距，故每次新建网页布局时应先清除一下所有网页元素的内外边距，此处通常用到通用选择器：  

```
* {  
	padding: 0;  
	maggin: 0;  
}
```

以上通常为所有CSS文件的第一段代码。
