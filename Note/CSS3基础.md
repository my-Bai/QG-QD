# CSS3基础  

> *CSS3为最新的CSS标准，所有浏览器目前均已兼容。*  

## 组合选择器  

组合选择器是指使用组合器连接两个及以上选择器的选择器，组合器定义了元素之间的关系，该选择器分为以下四种：  

- 后代选择器  
- 子选择器  
- 相邻兄弟选择器  
- 通用兄弟选择器  

此处讲解前两种：后代选择器与子选择器。  

### 后代选择器  

又称包含选择器，组成器为空格（SPACE），可以选择一个父元素中的所有某类别子元素，无论是直接子元素还是间接子元素都包括在内，其格式为`父元素 子元素 {声明}`。  

> *注意：选择包含间接子元素在内的某类别子元素时，推荐格式为 `父元素 直接子元素 1层间接子元素 ··· n层间接子元素 {声明}`。*  

### 子选择器  

组成器为大于号（`>`），可以选择一个父元素中的某类别直接子元素，间接子元素不包含于被选中元素中，其格式为`父元素>子元素 {声明}`。  

> *注意：父元素与子元素的选择可以使用任意基础选择器，如 `.abc #def {声明}`。*  

## 选择器列表  

选择器列表是选择器的一种，可以同时选择多种元素，通常用于集体声明，元素间用逗号（`,`）隔开，格式为：  

```  
元素1,
元素2,
···,
元素n {声明}
```

## 伪类选择器  

伪类选择器用于选择具有特定状态或位置的元素，选择时应先输入一个冒号（`:`），此处讲解其中一种：动态伪类选择器。  

### 动态伪类选择器  

- `a:link`：选择所有未被访问的链接  
- `a:visited`：选择所有已被访问的链接  
- `a:hover`：选择鼠标指针位于其上的链接  
- `a:active`：选择活动链接（鼠标指针按下之后与弹起之前）  

> *注意：实际应用时为确保生效，以上四种选择器最好按照从上到下的顺序进行代码布局。*  

> *注意：由于链接`<a>`元素在浏览器中具有独立的默认样式，故链接所需样式需要单独指定。*  

- `input:focus`：选择获得焦点的表单元素  

## 盒子模型  

### 组成部分  

CSS盒子模型的主要组成成分包括边框（`border`)，内、外边距(`padding`、`margin`）与实际内容（`content`）四部分。  

> *注意：边框与内、外边距会影响盒子模型的实际宽度与高度。*  

### 关于单行文本垂直居中  

使盒子的高度等于行间距，即可使盒子中的单行文本垂直居中。  

### 关于显示模式的转换  

某些情况下，一种显示模式的元素需要另一种显示模式的特点，称为显示模式的转换，以下为三种的转换模式：  

- 转换为块级元素：`display: block;`  
- 转换为行内元素：`display: inline;`  
- 转换为行内块元素：`display: inline-block;`

### 边框    

#### 样式  

`border-style`属性定义了要显示的边框类型，以下为其属性值：  

- 无边框（默认）：`none`  
- 隐藏边框：`hidden`  

> *注意：无边框与隐藏边框都看不见边框存在，但隐藏边框占据布局空间且影响盒子模型运算。*  

- 实线边框：`solid`  

- 虚线边框：`dashed`  

- 点线边框：`dotted`  

- 双边框：`double`  

- 3D边框：  

  - 坡口边框：`groove`  
  - 脊线边框：`ridge`  
  - 凹边框：`inset`  
  - 凸边框：`outset`  

  > *注意：3D边框兼容性严重，建议谨慎使用。*  

边框属性值可以为1~4个，顺序依次为：  

- 1个：同时定义四边。  
- 2个：先定义上下两边，再定义左右两边。  
- 3个：先定义上边，再定义左右两边，最后定义下边。  
- 4个：按照顺时针顺序从上边开始定义。  

> *注意：边框样式是所有边框属性的基础，若缺少样式则其他边框属性将不起任何作用。*  

#### 宽度  

`border-width`属性定义了边框四边的宽度，单位有像素（px），点（pt），派卡（pc），毫米（mm），厘米（cm），英寸（in）等，页面最常用单位为px，属性值可以为1~4个，定义顺序与`border-style`相同。

#### 颜色  

`border-color`属性定义了边框四边的颜色，属性值可以为1~4个，定义顺序与`border-style`相同。  

> *注意：以上三种属性还可为指定某边定义，格式为：`border-top/bottom/right/left-xxx`。*  

#### 属性简写  

如需缩短代码，可以在一个`border`属性中指定上述所有边框属性，其输入顺序为：  

1.`border-width`  

2.`border-style`  

3.`border-color`  

也可以只为一边进行属性简写，属性名格式为`border-top/bottom/right/left`

其余事项与背景属性简写相同。  

#### 表格  

`border-collapse`属性控制表格中相邻单元格边框的合并，合并时属性值应定义为`collapse`，单元格的合并指的是去除相邻边框的宽度叠加问题，使相邻单元格边框相遇后自动合并为一个边框，而非两个边框同时存在。  

> *注意：CSS定义表格边框时，选择器选中的元素应为`th`或`td`，若选中的元素为`table`，则只有整个表格的最外侧存在边框，无法起到划分单元格的作用。*  

#### 圆角  

`border-radius`属性定义了元素的边框圆角，属性值为圆角的水平与垂直半径，有如下两种表示方法：  

- 数字+单位  
- 百分比：表示水平半径为宽度的百分之几，垂直半径为高度的百分之几。  

该属性的属性值可以为1~4个，顺序依次为：  

- 1个：同时定义四角。  
- 2个：先定义左上角和右下角，再定义右上角和左下角。  
- 3个：先定义左上角，再定义右上角和左下角，最后定义右下角。  
- 4个：按照顺时针顺序从左上角开始定义。  

> *注意：`border-radius`属性还可为指定某角定义，格式为：`border-top/bottom-right/left-radius`。*  

### 内边距  

`padding`属性定义了内边距，即边框与内容之间的距离，格式为数字+单位，属性值可以为1~4个，定义顺序与`border-style`相同。    

> *注意：`padding`属性还可为指定某边定义，格式为：`padding-top/bottom/right/left`。*  

### 外边距  

`margin`用于在已定义的边框外为元素周围创建空间，属性值表示圆角半径，以数值表示时单位与`border-width`相同，属性值可以为1~4个，定义顺序与`border-style`相同。  

> *注意：以数值表示时其属性值还可以是负值，用于使元素向对应方向移动一定距离。*  

`margin`属性值也可以设置为`auto`，即浏览器自动计算外边距，使元素在其盒模型中水平居中，占据指定的宽度并将剩余空间在左右边界之间平均分配。

> *注意：`margin`属性还可为指定某边定义，格式为：`margin-top/bottom/right/left`。*   

> *注意：行内元素为照顾浏览器的兼容性，尽量只设置左右的内、外边距，不要设置上下的内、外边距。*  

### 关于内外边距  

#### 内边距与宽度、高度  

内边距与盒子模型的宽度（`width`）与高度（`height`）属性并不相容，如盒子宽度（`width`）定义为为20px，而左右内边距皆定义为为5px，则盒子模型的真正宽度应为20+5+5=30px，但若盒子模型的宽度或高度属性值未直接设置，则此时相应方向的内边距不会影响该盒子模型的宽度或高度。  

#### 外边距与水平居中  

外边距可以使盒子模型在页面中水平居中，只需要满足以下两个条件：  

- 盒子模型有指定的宽度（`width`）  
- 盒子模型的左右外边距属性值均为`auto`  

#### 外边距合并  

- 上下相邻的两个盒子模型相遇时，若上面的盒子模型带有下外边距，且下面的盒子模型带有上外边距，则外边距不会像内边距一样叠加，而是以其中数值更大的一方为二者间的真实外边距。  

- 父元素与子元素若同时具有上外边距或同时具有下外边距，则会发生嵌套元素塌陷——上外边距或下外边距将会选择二者中数值更大的一方为真实外边距，同时父元素与子元素会绑定，并同时根据该外边距位移，使父元素与子元素上边框或下边框间无间隔距离。  

  > *若想要父元素与子元素上、下边框间存在间隔距离，有以下三种方法：*  
  >
  > - *为父元素定义上、下边框*  
  > - *为父元素定义上、下内边距*  
  > - *为父元素添加属性`overflow: hidden`*  

  > *注意：浮动，固定或绝对定位的盒子模型不会发生嵌套元素塌陷。*  

#### 清除内外边距  

很多网页元素带有默认的内外边距，故每次新建网页布局时应先清除一下所有网页元素的内外边距，此处通常用到通用选择器：  

```
* {  
	padding: 0;  
	maggin: 0;  
}
```

以上通常为所有CSS文件的第一段代码。  

### 盒子阴影  

`box-shadow`属性可以为盒子模型添加阴影，格式为`box-shadow: h-shadow v-shadow blur spread color inset;`，以下为各个属性值的含义：  

- `h-shadow`：水平阴影的位置，代表阴影在x轴上的坐标值，可为负数。  

- `v-shadow`：垂直阴影的位置，代表阴影在y轴上的坐标值，可为负数。  

- `blur`：模糊距离，代表阴影的虚实程度，数值越小，阴影越清晰。  

- `spread`：阴影的尺寸，可为负数。  

- `color`：阴影的颜色。  

  > *阴影颜色一般设置时要加上透明度，多使用RGBA值。*  

- `inset`：阴影初始属性值默认为外部阴影（`outset`），此处为更改为内部阴影。  

> *注意：以上六种属性值中，前两种为必需项，后四种不做要求。*  

## 关于文本阴影  

`text-shadow`属性可以为文本添加阴影，格式为`text-shadow: h-shadow v-shadow blur color`，属性值含义和要求与盒子阴影相同。  

## 关于传统网页布局  

传统网页布局有三种方式： 

- 标准流：又称普通流或文档流，即标签按照规定好的默认方式进行排列，为最基本的布局方式。    

  > *默认方式指元素显示模式。*  

- 浮动  

- 定位  

> *注意：实际开发中，一个页面中三种布局方式通常同时存在。*  

## 浮动  

浮动可以改变元素默认的显示模式，可以使盒子模型同行显示且中间不留缝隙，`float`属性用于创建浮动框，将其移动到一边，直到左边缘或右边缘触及包含块或另一个浮动框的边缘，其属性值有以下三种：  

- `none`：元素不浮动（默认）  
- `left`：元素向左浮动  
- `right`：元素向右浮动  

### 浮动特性  

- 浮动元素会脱离标准流，又称脱标。  

  > *浮动的盒子模型不再占有原先的位置，其后的标准流元素可在其下重叠。*  

- 浮动元素会一行内显示，且元素顶部对齐。  

- 浮动元素具有行内块元素的部分特性。  

  > *任何元素都可以浮动。*  

> *注意：浮动元素常与标准流父元素搭配使用，且一个子元素浮动后，其他兄弟元素也应一起浮动。*  

### 清除浮动  

清除浮动的本质是清除浮动元素造成的影响，如未定义高度的父元素无法通过浮动的子元素来界定元素高度等，`clear`属性用于清除浮动，其常用属性值有：  

- `left`：清除左侧浮动的影响  
- `right`：清除右侧浮动的影响  
- `both`：同时清除左右两侧浮动的影响  

> *注意：实际开发中几乎只用 `both`。*  

清除浮动有四种方法，分别为：  

- 额外标签法：又称隔墙法，指在最后一个浮动的盒子后添加一个带`clear`属性的空标签，缺点为在一定程度上影响网页代码布局。  

  > *注意：空标签所对应元素必须是块级元素。*  

- 父元素添加`overflow`属性：为父元素添加`overflow: hidden`，缺点为无法显示子元素溢出盒子模型的部分。  

- 父元素添加`after`伪元素：在父元素的`class`属性中添加一个`clearfix`属性值，然后在CSS文件中添加以下代码：  

  ```
  .clearfix::after {  
      content: "";  
      display: block;  
      clear: both;  
  }  
  ```

  这种方法不需要添加额外的元素标签，在一定程度上简化了网页代码布局。  

- 父元素添加双伪元素：类似上一方法，在父元素的`class`属性中添加一个`clearfix`属性值，然后在CSS文件中添加以下代码：  

  ```
  .clearfix::before,  
  .clearfix::after {  
      content: "";  
      display: table;  
  }  
    
  .clearfix::after {  
      clear: both;  
  }  
  ```

## 定位  

定位的作用是让盒子模型在父元素内自由移动，或页面滚动时使其固定于页面某处不动，定位由以下两部分组成：    

### 定位模式  

用于指定一个元素在文档中的定位方式，`position`属性用于设置元素的定位模式，以下为其最常用的四个值：  

- `static`：静态定位，指无定位（默认）。  

- `relative`：相对定位，指相对该元素的原位置进行定位。  

  > *注意：与浮动不同，元素相对定位后，所占空间仍为其未定位前的原位置空间。*  

- `absolute`：绝对定位，指相对该元素的最近一级的已定位父元素进行定位。  

  > *若无父元素，或父元素未定位，则以浏览器为准进行定位，若存在更高级别有定位的父元素，则以该父元素为准。*  

  > *注意：与相对定位不同，元素绝对定位后，不再占有未定位前的原位置空间（脱标）。*  

- `fixed`：固定定位，指元素固定于浏览器可视区的位置，使浏览器页面滚动时，该元素的位置不变，其参照点为浏览器的可视窗口。  

  > *注意：类似相对定位，元素固定定位后，不再占有未定位前的原位置空间（脱标）。*  


### 关于子绝父相  

若子元素为绝对定位，则父元素应为相对定位，此举是因为父元素需要占有位置，而子元素不需要。  

> *注意：若父元素不需要占有位置，则也可以为绝对定位。*  

### 边偏移  

用于决定元素的最终位置，该部分含以下四种属性：  

- `top`：顶端偏移量，定义元素相对其上边线的距离。  
- `bottom`：底部偏移量，定义元素相对其下边线的距离。  
- `left`：左侧偏移量，定义元素相对其左边线的距离。  
- `right`：右侧偏移量，定义元素相对其右边线的距离。  

> *边偏移属性值可以是数字+单位，也可以是百分比。*  

### 叠放次序  

`z-index`属性用于控制已定位盒子模型的叠放次序，即从最底部到最顶部的等级顺序，属性值正负均可，默认为`auto`（即后来的在先来的之上），属性值越大，元素越靠上。  

> *注意：属性值后不能加单位！*  

### 关于绝对定位盒子模型的居中  

1.使用`left: 50%`或`top: 50%`，使盒子模型的左侧或上侧与页面水平或垂直中线重合。  

2.使用`margin-left: xxpx`或`margin-top: xxpx`，使盒子模型向左或向上平移半个盒子模型的宽度或高度。  

### 关于定位的特殊特性  

- 行内元素添加了绝对定位或固定定位后，可直接设置宽度与高度。  

- 块级元素添加了绝对定位或固定定位后，其默认大小为内容大小。  

- 脱标的盒子模型不会触发外边距合并的相关问题。  

- 绝对定位或固定定位的盒子模型脱标后会压住其后的标准流元素内容。  

  > *注意：浮动脱标后不会压住其后的标准流元素内容。*
